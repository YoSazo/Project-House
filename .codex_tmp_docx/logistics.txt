1. Purpose
The Logistics Robot solves the material transport problem on a live, uncontrolled build site — specifically the compounding failure where cheap transport becomes unreliable when terrain, timing, payload sensitivity, and path conflicts all interact simultaneously.

3. C2A Transmutation Chain
Nine constraints were identified and transmuted. Each one was derived from first principles before any engineering solution was proposed.

Transport looks simple until the environment is messy and the load is timing-sensitive. A cheap robot on a live build site faces uneven terrain, dynamic obstacles, changing destinations, payload instability, and path conflicts with other robots.

4. Refinements Added in This Version
The following improvements were added during spec refinement beyond the original transmutation chain.

The original spec assigned a dedicated rolling verifier robot. This creates a single point of failure — if that robot breaks, the relay chain loses its verification layer.

4.3 Lane Graduation
The original spec treats lanes as either conditioned or not. In reality, lanes improve gradually and payloads have different sensitivity levels — a pile of sand is not the same as a stack of compressed earth blocks waiting for placement.

Lanes are assigned a condition score from 0 to 1 based on swarm consensus data. The House Brain routes payloads accordingly — fragile or precision components only travel on lanes scoring above 0.85, bulk materials can travel on lanes scoring above 0.5, and empty robots can travel on any lane above 0.2. This lets the site become productive faster because bulk logistics can begin before the lanes are fully conditioned.

4.4 Congestion Avoidance
Multiple robots on the same lane in the same direction at the same time creates queueing, which reduces utilization and can cause payload instability from stop-start motion.

The House Brain assigns time-slotted dispatch windows per lane segment. A robot is not released from the pickup zone until its lane segment has a clear slot. Slot length is calculated from lane length divided by robot speed plus a buffer. This is identical to how airport ground traffic control works — a simple queuing protocol removes most congestion without requiring robots to communicate directly with each other about position.

5.3 Body-as-Gauge Mechanism
The chassis can be lowered on the vertical axis to bring a reference contact frame close to the terrain surface. This is the primary geometric verification tool — no additional sensors required.

Vertical travel: 60 – 120 mm downward from standard operating height
Contact frame: rigid perimeter rail at the lowered position, contacts terrain at 4 points
Contact sensing: load cells or current-draw monitoring on lowering actuators
Self-leveling: the contact frame is gimbaled to match local slope up to ±8 degrees
Speed during gauge check: robot stops completely before lowering

5.4 Payload System
Flatbed tray with modular retention rails — adapts to blocks, tool kits, and bulk material
Payload weight sensing via tray load cells
Latch confirmation sensor — positive lock/unlock state always known
Dock alignment guide — passive funnel geometry assists handoff without active alignment robot
Max payload: sized to carry one full compressed earth block batch per trip

Design principle: every sensor serves multiple purposes. The IMU serves both navigation and lane quality scoring. The actuator current serves both lowering control and geometric verification. No sensor exists for a single use.

7. Lane Doctrine
7.1 Reference Patch Protocol
The reference patch is established once at the start of site operations and is the single source of geometric truth for all lane conditioning.

Location: a pre-selected flat area near the main staging zone, manually prepared to approximate target grade
Stamped by: Survey robot during site qualification, or mother logistics bot before site operations begin
Recorded as: target blade force band, target body vibration band, target tilt stability, target wheel-slip band
Protected by: House Brain marks as no-heavy-load zone, robots navigate around it except during verification passes
Re-stamped when: verification consensus shows drift greater than 15% from baseline

7.3 Relay Protocol
What gets relayed: target blade force band + target body-gauge signature + current lane condition score per segment.

How often: every robot that crosses the reference patch republishes. Robots ahead receive an update on each delivery cycle. Front robots stop extending if their local reading deviates more than 20% from the last relayed reference without explanation.

Conflict resolution: if two relayed values disagree by more than threshold, the robot closer to the reference patch wins. The House Brain logs the disagreement for inspection.

13. Cost Direction
This is not a final BOM. The design intent is to keep every component as cheap as possible by putting intelligence into swarm behavior rather than individual robot hardware.

Cheap wheeled base — agricultural or industrial rover chassis, not custom
Simple payload geometry — standardized tray sizing to match compressed earth block dimensions
One weak grading tool — drag blade only, no active earthmoving
No humanoid complexity — no arms, no legs, no balance systems
Motor impedance sensing as force proxy — avoids dedicated force-torque sensors
Swarm truth over individual sensor sophistication — cheap sensors made reliable through consensus

14. Success Criteria
The Logistics Robot is successful when it can:

Move payloads reliably from source to destination on partially conditioned terrain
Improve recurring paths measurably over repeated trips without overbuilding them
Match lane segments to a shared grade reference without central coordination per robot
Preserve lane alignment over distance using relay truth rather than single-robot memory
Distribute the verification role across the swarm with no dedicated verifier robot
Hand off materials without requiring precision alignment infrastructure at each drop zone
Emit usable transport and lane telemetry to the House Brain in real time
Continue operating in degraded mode when individual robots fail or relay truth is stale
