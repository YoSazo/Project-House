4. Constraints and Your Transmutations

Constraint 1 â€” The Needle-in-a-Hole Problem
Problem:

Transmutation: Coordinate + Insertion Logic, Not Fingers

Constraint 2 â€” The Vision Cost Problem
Problem: 
If the robot must "see" the whole house like a human, cost and complexity explode.

We don't use vision, we use address logic. The house is a grid. The brain simply says square 3x9, 3x10 need this. No vision required.
Result:
House Brain knows which cells are empty/filled
Robot receives discrete target coordinates
UWB/grid positioning gets robot to the cell
The house becomes a coordinate-addressed build space

Constraint 3 â€” The Numb Fingers Problem
Problem: 
The robot reaches the correct coordinate but still cannot feel whether the part is landing correctly.

The same mechanism is applied to depth. The grid is not only horizontal, but depth-wise too â€” thus it can feel tension by measuring how the block goes in.
Result:
Expected insertion path is known
Expected Z travel is known
Any slowdown, resistance, or path deviation becomes the force/touch proxy
No fingertip sensors required â€” this is the core Assembly breakthrough

Constraint 4 â€” The Blind Push Problem
Problem: 
If the robot keeps pushing after deviation begins, it may crack the part or damage the surrounding structure.
Transmutation: Deviation Triggers Stop, Not Force
If the path through depth is wrong, the robot knows it is wrong because the insertion through Z is no longer matching the expected path.
Result:

Micro-adjust
Retry
Only continue when descent resumes cleanly
System becomes: expected corridor â†’ deviation detected â†’ micro-adjust â†’ resume

Constraint 5 â€” The Expensive Touch Problem
Problem:

Transmutation: Motion Deviation Is Touch
The same mechanism applied to depth replaces that. The house/brain tells it what should happen, and when the block doesn't go through that path correctly, that itself is the tension signal.
Result:

Depth deviation acts as touch
Motor effort / impedance proxy acts as touch
Insertion-rate mismatch acts as touch
No costly fingertip skins required
